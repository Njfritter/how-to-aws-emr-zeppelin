What we'll be making:

Zeppelin as a user interface for coding Spark, run on AWS EMR with multiple AWS EC2 machines as a cluster, accessing public data hosted on AWS S3.

AWS web interface
AWS CLI

First things first,
log into your account.

We're in. Let's first make a keypair so you can securely connect to the otherwise hidden amazon cloud.



  • Go to the EC2 key-pair UI.
  • Click "Create Key Pair."
  • Enter a name.
  • Create it.
  • Download the pem file.
  • We'll use this later.

We're logged in, and we have a keypair to connect with. Now we need to actually create the cluster.
(AKA the orange stuff.)



  • Navigate to the EMR UI.
  • Click "Create Cluster".

Name the cluster.

Choose Spark + Zeppelin.

Keep the defaults for now.

It is suggested to provide memory-optimized core instances for later revisions of your cluster.

Use the keypair we previously created, then click "Create Cluster".

That's it. You've created the cluster. Don't worry, it's natural to have to wait about 10 minutes for everything to provision.

Terminate your cluster when you're done. You're spending money right now.
